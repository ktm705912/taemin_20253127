## 1. 전체 명령어 간단 설명

+ `top` — 실시간 시스템 모니터링할때 사용. 
+ `ps` — 프로세스 조회할때 사용.
+ `jobs` — 백그라운드 작업 확인할때 사용.
+ `kill` — 프로세스 종료할때 사용.

# 2. 명령어 상세 설명

## 2-1 `top` — 실시간 프로세스 모니터링

### 사용 방법
```bash
top
```

### 기능
- CPU, 메모리 등의 시스템 상태를 실시간으로 확인함.
- 서버 운영 시 가장 많이 쓰이는 명령어 중 하나이다.

### 옵션(단축키)
+ `P` : CPU 사용률 기준 정렬  
+ `M` : 메모리 기준 정렬  
+ `k` : 프로세스 종료  
+ `h` 또는 `?` : 도움말  
+ `q` : 종료  

### 출력내용
| 필드 | 설명 |
|------|------|
| PID | 프로세스 ID |
| USER | 실행 사용자 |
| %CPU | CPU 사용률 |
| %MEM | 메모리 사용률 |
| TIME+ | 누적 CPU 시간 |


---

## 2-2 `ps` — 프로세스 정보 조회 (kill 사용 시 필요 (pid))

### 사용 방법
```bash
ps
```

### 기능
- 현재 실행 중인 프로세스 정보를 스냅샷 형태로 제공한다.
- `grep`과 함께 특정 프로세스 검색 시 유용하게 사용가능하다.



### 옵션
| 옵션 | 설명 |
|------|------|
| -e | 모든 프로세스 출력 |
| -f | 풀 포맷 출력 |
| aux | BSD 스타일 전체 프로세스 출력 |
| -ef | 전체 프로세스를 상세하게 출력 |


---

## 2-3 `jobs` — 백그라운드 / 중지 작업 관리

### 사용 방법
```bash
jobs
```
### 기능
- 현재 셸에서 실행된 백그라운드 작업을 확인함.

### 옵션
| 옵션 | 설명 |
|------|------|
| -l | PID 포함 |
| -p | PID만 출력 |
| -s | 중지된 작업만 표시 |

### 예시
```bash
sleep 10 &
jobs
```


---

## 2-4 `kill` — 프로세스 종료

### 사용 방법
```bash
kill <PID>(process id)
```

### 기능
- 프로세스에 시그널을 보내 종료 또는 제어함.


###  옵션
| kill | -번호 | 설명 |
|--------|------|--------|
| kill | -15 | 정상 종료 요청 |
| kill | -9 | 강제 종료 |
| kill | -19 | 프로세스 정지 |
| kill | -1 | 재시작 또는 설정 갱신 |

### 예시
```bash
kill -9 taemin #taemin 강제종료
```
